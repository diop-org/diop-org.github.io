TimThumb Image Resizing Script - Common Problems - top

  1. !!! READ ONLY IF THE IMAGES YOU PERSONALLY INSERTED INTO THE SLIDER OR YOUR POST ARE NOT SHOWING UP !!!
  2. TimThumb will automatically create a /cache/ subfolder with proper write-permissions. If your host server doesn’t allow this by default, be sure to manually change the /framework/scripts/timthumb/cache/ folder permissions to 777. If you don't know how to change permissions, google: change permissions + "name of your FTP application"
  3. If neccessary, make sure the folder which contains TimThumb is also set to 777 as well.
  4. Please note that on some hostings you need to set 755 instead of 777.
  5. Please note that if everything above fails, you need to contact your hosting provider to white-list timthumb script
  6. Please note that this is not theme's issue. So please refer to your hosting providers - if they have a good support, they should know what you want, straight away
